Section 1 · 프로젝트 이해

이번 프로젝트를 이렇게 이해했습니다

🎯

콘텐츠에서 구매까지 끊김 없는 전환

미국 인플루언서의 링크, 캠페인, 큐레이션 콘텐츠에서 유입된 사용자가 한국어 상품 탐색, 장바구니, 결제, 배송 조회까지 한 흐름으로 이어져야 합니다.

📊

어트리뷰션과 운영 데이터의 일치

어떤 인플루언서 링크가 매출과 재구매를 만들었는지, 할인/환불/취소가 반영된 실매출 기준으로 추적하고 정산까지 연결해야 합니다.

🏪

벤더 운영과 백오피스가 동시에 필요

브랜드/벤더는 상품, 재고, 출고, 클레임 상태를 관리하고, 운영 관리자는 승인, 품질 검수, 프로모션, CS, 예외처리를 통합 관리할 수 있어야 합니다.

✅

이번 제안의 기준은 Full Scope

검증용 축소 MVP가 아니라, 런칭 가능한 운영 버전 기준으로 소비자·인플루언서·벤더·관리자·정산 전 모듈을 함께 제안합니다.

Section 1 · 목표 정의

프로젝트 목표

1

인플루언서 유입의 매출화

콘텐츠 링크, 전용 쿠폰, 캠페인 페이지를 통해 실제 주문과 재구매까지 연결합니다.

2

벤더/관리자 운영 효율화

상품 등록, 재고 반영, 출고, 클레임, 승인, 운영 예외처리를 한 플랫폼에서 처리합니다.

3

정산의 투명성과 재현 가능성 확보

판매액, 할인, 취소, 환불, 수수료, 프로모션 비용을 반영한 정산 원장을 제공합니다.

4

런칭 후 운영 가능한 상용 수준 완성도

개발 완료 시점에 기능만 있는 상태가 아니라 운영팀이 실제로 사용할 수 있는 관리 체계를 함께 제공합니다.

Section 2 · Full Scope

제안 범위 전체 구성

이번 제안 범위는 프로덕션 런칭 가능한 전체 범위입니다. 소비자 여정만 구현하는 축소안이 아니라, 플랫폼 운영에 필수적인 벤더/운영/정산 모듈을 동시에 포함합니다.

Section 3 · Architecture

권장 아키텍처

🌐

Experience Layer

Next.js 기반 소비자 웹, 인플루언서 랜딩, 벤더 콘솔, 관리자/정산 콘솔을 역할별로 분리합니다.

⚙️

Core Domain Layer

상품, 가격정책, 주문, 프로모션, 클레임, 정산 규칙을 NestJS 기반 도메인 서비스로 구성합니다.

📦

Data & Integration Layer

PostgreSQL, Redis, Queue, S3/Blob, PG/택배/메시징/Analytics 연동을 통해 운영 안정성과 확장성을 확보합니다.

Consumer Web
상품 탐색 · 결제 · 마이페이지
→
Core APIs
Catalog · Order · Claim · Settlement
→
Admin / Vendor
운영 · 공급 · 정산 관리
Frontend
Next.js + TypeScript

SEO 대응 랜딩, 관리 콘솔, role-based UI

Backend
NestJS + PostgreSQL

도메인 모듈, 감사 로그, 정산 원장, Admin API

Infra
Vercel + AWS

CDN, storage, queue, cron, monitoring, backups

Integrations
PG / 배송 / 메시징 / Analytics

국내 결제수단, 배송 추적, 알림, 유입/매출 분석

  • ✓

정산 공식 예시

정산 대상 금액
실결제금액 - 쿠폰/프로모션 부담금 - 환불/취소 반영 - PG/운영 수수료 - 인플루언서 수수료 + 정산 조정금

정산 관리자에서 파트너 유형별 규칙을 관리하고, 주문 원장과 지급 원장을 분리하여 재계산과 감사 추적이 가능하도록 설계합니다.

Section 5 · Implementation Plan

구현 접근 방식

Track 1

Product Definition & UX

요구사항 정리, 역할/권한 설계, IA, 핵심 화면 와이어프레임, 관리자 운영 흐름 확정.

Track 2

Commerce Core Build

상품/가격/주문/클레임/프로모션 도메인과 소비자 구매 플로우를 병렬 개발합니다.

Track 3

Operations & Settlement

벤더 콘솔, 운영 관리자, 정산 원장, 지급 리포트, 감사 로그를 함께 구성합니다.

Track 4

Launch Readiness

QA/UAT, 테스트 데이터, 운영 매뉴얼, 배포/모니터링 세팅까지 마무리해 실제 운영 전환이 가능하도록 준비합니다.

실행 원칙

1. 역할 분리
소비자/인플루언서/벤더/관리자 권한을 명확히 분리합니다.
2. 운영 우선
정산과 예외 처리까지 초기 범위에 포함합니다.
3. 데이터 일관성
주문 원장과 정산 원장을 함께 설계합니다.
4. 런칭 품질
QA/UAT와 운영 문서까지 납품 기준에 포함합니다.

Section 8 · Risks & Support

리스크와 대응, 그리고 운영 지원

주요 리스크와 대응

  • 인플루언서 캠페인 급증 시 재고 불일치
